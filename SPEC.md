# spec2c — JSON Spec → Vehir-pattern C Tool

Generate correct-by-construction C tools from a declarative JSON specification.
The generator emits the Vehir skeleton — a proven, safe pattern used by all
Vehir tools (db, pain, file-age, forge, mail, git-identity).

## Why

AI-written C tends to produce: memory leaks, unchecked returns, suppressed output,
SQL injection via string interpolation, hardcoded paths. These are MECHANICAL
errors — the kind a template eliminates by construction.

A JSON spec declares WHAT the tool does. The generator emits the skeleton.
The skeleton guarantees: no leaks, no unchecked returns, no suppressed output,
no SQL injection, no hardcode. A hand-written C slot handles tool-specific logic.

## Spec format (draft)

```json
{
  "name": "my-tool",
  "description": "Does something useful",
  "commands": {
    "default": {
      "description": "Runs the tool",
      "inputs": ["file_path"],
      "db_queries": [],
      "output": { "ok": "bool", "result": "string" }
    }
  },
  "config_keys": ["DATABASE_URL"],
  "core": {
    "file": "my_tool_core.c",
    "function": "my_tool_run"
  }
}
```

## Generated skeleton

```c
// Generated by spec2c. DO NOT EDIT.
// SPDX-License-Identifier: Apache-2.0
#define _POSIX_C_SOURCE 200809L
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <cjson/cJSON.h>

// Broker config reader — generated
static char *cfg_get(const char *key) { /* safe fopen + parse */ }

// DB query wrapper — generated, bind-params
static cJSON *db_query(vehir_db *db, const char *sql, const char **params, int n) { /* safe */ }

// Error output — generated, JSON
static _Noreturn void die(const char *msg) { /* safe */ }

// Tool-specific logic — hand-written in separate file
extern int my_tool_run(vehir_db *db, int argc, char **argv);

int main(int argc, char **argv) {
    if (argc > 1 && strcmp(argv[1], "--help") == 0) {
        printf("Usage: my-tool [file_path]\nDoes something useful\n");
        return 0;
    }
    vehir_db *db = vehir_db_open();
    if (!db) die("db-proxy unreachable");
    return my_tool_run(db, argc, argv);
}
```
